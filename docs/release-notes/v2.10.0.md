# jawata v2.10.0 — Sprint 23: runtime grounding (run tests · prove coverage · gate on evidence) + parity closed

The biggest functional release since the plain-Maven switch: jawata now RUNS
the project's tests itself on every project shape, PROVES what they exercise
(coverage with honest provenance, who-tests-what, targeted mutation), and
GATES changes on that evidence — advisory-first, mechanical refactorings
exempt. Plus every competitive-parity item from the Sprint-23 landscape diff.
0 net new top-level tools — everything lands behind existing front doors.

## D1 — run_tests works, on every shape (the execution spine)

- Forked runner JVM (JUnit-platform launcher shipped in dist `tools/`), the
  project's OWN classpath from the importer — never the server's. JUnit 4/5 +
  TestNG detected from the classpath; method/class/package scopes.
- §13 safe execution: cleared environment + allowlist, `-Xmx512m` default,
  session tmpdir, explicit timeout → process-TREE reap (no orphan JVMs),
  bounded 1 MB/100-line stream tails, max 4 concurrent sessions, and an
  honest `evidenceFinalized=false` on crash/kill/timeout — never invented
  totals.
- Async: `action=start/status/cancel` — per-class progress events observable
  BEFORE completion; summaries survive completion (bounded ring).
- All shapes proven on fixtures: plain Maven, multi-module reactor
  (B-tests-exercise-A), Gradle, plain-Java launch descriptor, and PDE test
  bundles (below).
- All five historic `@Disabled` tests are ENABLED and green — the self
  Disabled-probe returns 0.

## D7 — PDE / MANIFEST-first classpath (external bundle pools)

`Require-Bundle`/`Import-Package` now resolve against EXTERNAL pools
(`jawata.bundle.pools` → `~/.p2/pool/plugins` → the server's own dist
bundles): manifests indexed once per pool (mtime-cached), highest version
wins, resolved jars appended as exported library entries. PDE fixture trees
compile 0 errors and their suites run through the same forked runner.

## D2 — the suite in a quarter of the time

Full local suite 1761 s → ~190 s (4 timing-balanced shards via
`build/run-suite.sh`) / ~330 s serial. The real lever: a Maven-classpath
cache keyed by a SHA-256 over every `pom.xml` in the tree — the `mvn`
shell-out was ~85 % of suite cost.

## D3/D4/D5/D6 — the coverage evidence chain (all behind `run_tests`)

- **Collector + artifact identity:** JaCoCo agent on the forked runner;
  artifacts = exec data + provenance manifest (git revision + dirty
  fingerprint, class/source roots, selection, evidence kind, versions,
  completion status) under the workspace state dir; explicit `delete`.
- **Honesty first:** EIGHT distinguishable line states (covered / missed /
  non-executable / generated-or-excluded / not-instrumented /
  instrumentation-failure / stale-bytes / unknown-run-failed); a report
  against modified class bytes is REFUSED as stale; same-FQN-in-two-bundles
  stays two facts; child-JVM work is marked outside-the-measurement — never
  silently zero.
- **Queries:** `coverage_report` / `uncovered` (with caller/implementation/
  existing-test links + a next-action hint), `delta` on working-tree/staged/
  commit-range diffs (rename-aware — a moved covered line is not a gap),
  named `baselines` + regression-vs-baseline, versioned threshold policy
  with always-visible waivers, two-run `stability` (unstable lines never
  count as covered), CI `import` (class-id checked), per-test attribution →
  `tests_covering` / `coverage_of_test` / `impacted_tests` ranked
  focused-first, and targeted PIT `mutation` (survivors mapped to symbols +
  candidate missing assertions).
- **The advisory gate:** a behavioral change with uncovered changed lines
  gets a done-time advisory naming them; transforms by the mechanical tools
  (rename/extract/move/format/…) NEVER trigger it. `find_quality_issue`
  gains the evidence-backed `coverage_lack` kind — stale or absent artifact
  = an honest "no fresh evidence", zero invented findings.

## D8 — source for ANY type by FQN

`inspect(kind=source)`: workspace source → attached source → the sibling
`…-sources.jar` (existing only, no silent fetch) → an honestly-labeled
disassembled stub. Works on Maven and PDE trees.

## D11 — call-hierarchy precision

A Javadoc `{@link}` is documentation, not a call — incoming call hierarchies
no longer show phantom `<initializer>` callers.

## D9 — `fields` per-row projection (the C13 decision)

The measured list-row redundancy was ~2.5–3× (restated qualified names,
per-row paths). The 5 list-heavy tools — `search_symbols`,
`find_references` (all kinds), `get_call_hierarchy` (both directions),
`find_pattern_usages`, `get_diagnostics` — accept
`fields=["filePath","line"]` and return rows with exactly those keys.
No `fields` = full rows (fully back-compatible); malformed `fields` is an
invalid-parameter error, never a silently-full response. A pipe-style query
DSL was evaluated against live token measurements and rejected — projection
captures ~75 % of the saving at none of the language cost.
