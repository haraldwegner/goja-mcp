# GOJA v2.6.0 ‚Äî pre-launch completeness

The completeness release: the composition-axis refactoring primitives, two correctness
CRITICALs, full client parity (Cursor), the measurement-tooling gaps, and the
detect‚Üícure recipe bridge ‚Äî everything the Java loop needs to stand on its own.

**Tool surface: 41 ‚Üí 43** (two net-new front doors; everything else folds into existing
`kind`/`action` parameters ‚Äî no new framework).

## Composition-axis refactoring (the axis GOJA didn't cover)
- **`move_method`** *(new front door)* ‚Äî Move Instance Method: relocate a method onto the
  type of one of its parameters/fields, rewriting every call site. Parity-gated, reversible.
- **`change_visibility`** ‚Äî a mode on `change_method_signature`: widen/narrow a method's
  visibility in one call, with a reference-impact report.
- **`retarget_call_sites`** ‚Äî a mode that rewrites call sites onto a new delegate/target.

## Correctness
- **Empty-buildSpec fix** ‚Äî goja-synthesized projects now carry the Java builder, so
  `compile_workspace` / `get_diagnostics` report real compiler errors (the diagnostics
  path is no longer vacuous).
- **Frontmatter-quote loader fix** ‚Äî a quoted `symbol: "pkg.Type#member"` anchor now
  resolves and is recalled.
- **Index hygiene** ‚Äî `.claude/**` and `*.edit-bak` files are excluded from the source
  index (no phantom duplicate types in search / call-hierarchy).

## Measurement tooling
- **Composed encapsulation audit** (`analyze(kind=encapsulation)`) ‚Äî catches the
  setter-false-pass that `find_field_writes` alone misses: it unions external direct
  writers with external callers of state-mutating methods and reports the per-field
  poke-set count.
- **Dependency-direction rules** ‚Äî `find_quality_issue(kind=forbidden_edge)` reports a
  declared forbidden package edge.

## Ergonomics
- **`find_string_literals`** *(new front door)* ‚Äî AST string-literal / full-text search
  (find a log string's emitter).
- **FQN member addressing** on `get_call_hierarchy` (a `pkg.Type#member` cue, no
  file:line needed).
- **Baseline / trend diffing** for family sweeps (`{new, fixed, unchanged}` on a re-run).
- **Incremental-by-default build kind** for lower-latency freshness after edits.
- `search_symbols` wildcard fallback + tool-name alias hints.

## Detect ‚Üí cure ‚Üí execute
- **Recipe bridge** ‚Äî a `find_quality_issue` smell kind hands straight to
  `refactoring(action=plan, kind=<smell>)`, which resolves the cure recipe and returns a
  planId that `apply_plan` runs parity-gated.
- **Multi-catalog conflict seam** ‚Äî where ‚â•2 detectors flag one location, family sweeps
  surface the arbitration inputs (detectors + families + a prevalence signal) and decide
  nothing.
- **`experience(kind=fallback_report)`** ‚Äî a ranked capability-gap backlog aggregated from
  the recorded `goja-fallback` declarations.

## Client parity (with goja-studio)
- Cursor gets a managed `~/.cursor/hooks.json`: guard (`failClosed`) + `sessionStart`
  primer at full parity; recall (dual-cue + on-Edit-of-.java) as a side-effect (Cursor
  cannot inject model context on `beforeSubmitPrompt` ‚Äî the recalled fact reaches the
  model via the rule block + primer + MCP `meta.steering`); observer fire-and-forget.
