# jawata-mcp v3.2.1 — day-0 dogfood patch (Sprint 26)

Three defects found by working v3.2.0 in anger on its first day — two of
them caught by the user, one by the release's own honesty counter. All
three share one theme, the codebase's recorded top-bug class: **a degraded
result presented as a normal one.**

## No more silent store degrade (finding #1)

At the fleet flip, one resident's shared-store open failed transiently and
the old code silently fell back to a bare in-memory store: 367 memory-seed
entries served as if they were the 2001-entry database, detectable only by
a missing file size. Now a failed open hands back a **self-healing store**:

- every tool answer carries a DEGRADED notice naming the reason — nobody
  has to spot a missing file size again;
- the real database is retried every 30 seconds in the background;
- on success, everything recorded during the outage is **replayed** into
  the real store — the degraded window loses nothing — and the notice
  clears. A permanent failure keeps the notice up forever: loud beats stuck.
- `stats` reports `in-memory (DEGRADED: <reason>)`, never a bare in-memory.

## No more fake green on an empty workspace (finding #2)

A resident whose every workspace project failed to load reported
Ready/healthy and answered compile requests with success-and-zero-errors —
a no-op that read as a green build. Now the workspace watcher records
per-project load failures; an auto-load where every project failed flips
health to FAILED with the first cause and the remedy, and
`compile_workspace` on an empty workspace **refuses** with
`NO_PROJECTS_LOADED` — nothing compiled is not a green build.

## Learner-event writes retry (finding #3)

The learner label stream was dropping ~40% of events on the shared store:
the connection self-heals only at acquisition, so a statement dying
mid-write (auto-server handoff, lock timeout against a peer resident) lost
its event on the single attempt. Writes now retry up to 3 times on a
forced-fresh connection with backoff; an event counts as dropped only after
the budget — and the loud `failedWrites` counter that caught this stays.

Suites at release: sharded 1483/1483. 12 carriers bumped 3.2.0 → 3.2.1;
jawata-studio stays at v3.2.0 (no studio changes in this patch).
