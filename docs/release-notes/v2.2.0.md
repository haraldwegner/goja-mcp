# GOJA MCP v2.2.0 — the crawl finds everything (Sprint 21b, resident side)

Memory-simplification release: the store's maintenance becomes automatic and its
discovery becomes complete. The `experience` tool surface is unchanged (15 kinds).

## Autofind — memory discovery is user-level, like the store

`load`/`reseed` default roots now discover the usual agent-memory locations instead of
only this workspace's convention:

- **Every** existing `~/.claude/projects/*/memory` directory — your knowledge is
  user-level (shared store), so discovery is too, not limited to the current
  workspace's projects.
- Per project directory: `.cursor/rules/` (Cursor project rules — `.mdc` files are now
  crawled alongside `.md`), legacy `.cursorrules`, `AGENTS.md`,
  `.github/copilot-instructions.md`, `.windsurfrules`.
- The layered `CLAUDE.md` set (global → ancestors → project) as before; explicit
  `-Dgoja.memory.roots` entries still come first.

## The crawl finds everything

- `recursive` defaults to **true** for `load` and `reseed` (pass `recursive:false` to
  stay flat).
- The crawl caps are no longer tuning values but pure runaway backstops:
  depth 32 / 10 000 files / 256 MB (were 5 / 200 / 2 MB). `-Dgoja.memory.max*` remains
  honored; a fired backstop is still reported, never silent.

## Refresh is automatic

The staleness pass (`refresh`: re-resolve Java pointers, supersede dead ones) is
maintenance, not a user decision:

- runs once after project auto-load (async — zero startup cost),
- runs after every successful `load`, `reseed` and `import`; the report is merged into
  the response under `refresh`.
- Only `language='java'` anchors are judged (non-Java entries stay opaque, v2.1.0
  item I). The `refresh` verb remains available for prompt use.
