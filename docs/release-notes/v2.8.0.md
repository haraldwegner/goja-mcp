# JAWATA v2.8.0 — Eclipse 2026-06 target-platform re-base (Sprint 22c)

The platform pin inherited from the very first commit — Eclipse 2024-09 — is replaced
by a deliberate choice of the newest stable train, **Eclipse 2026-06 (Platform 4.40,
JDT Core 3.42 / compiler 3.46)**. Seven quarterly trains in one hop; behaviour is
identical, proven by the full suite and a live deploy. No new features; the payoff is
current JDT (parses Java 25/26 source — the runway for the Sprint-31 migration engine)
and a supported base for upstream participation.

## Platform

- **Target platform: Eclipse 2024-09 → 2026-06.** Zero compile breaks across the hop —
  the JDT/LTK/search internals we sit on were API-stable. Java floor **stays 21**
  (4.40 requires Java SE 21; no newer JDK anywhere in the build).
- **One JDT platform bug found, filed, and neutralized:
  [eclipse.jdt.core#5188](https://github.com/eclipse-jdt/eclipse.jdt.core/issues/5188).**
  Since ~4.35, JDT's preview-API warning machinery NPEs (unguarded, before any
  severity check) during search match locating when
  `jdk.internal.javac.PreviewFeature$Feature` is unresolvable — killing reference
  searches that touch preview-annotated Java 21+ APIs. Reported upstream by us;
  shipped here as a locally repacked
  `org.eclipse.jdt.core.compiler.batch 3.46.0-jawata5188` (the official jar with one
  null-guarded class swapped in; the `org.jawata.jdtpatch` module is the reproducible
  factory, EPL-2.0 notice included). A 4-test sentinel (`JreTypeResolutionTest`) pins
  the wiring and gates removal of the patch when a fixed train ships (4.41+).
- **Ported from upstream drift:** `get_diagnostics` no longer reports info-severity
  markers as warnings (filter applied on both the AST and marker paths).

## Pins (each carries a dated decision in the target file)

- **JUnit stays on the 2024-09 location** — trains from 2025-12 ship JUnit 6, which
  Tycho 5.0.x's junit5 surefire provider cannot run. Same split upstream chose.
  Revisit with the Sprint-23 test-runner work.
- **H2 kept at 2.2.224** — the user-shared AUTO_SERVER store is opened concurrently
  by deployed residents; mixed H2 versions on one live store risk corruption. Bumps
  move the whole fleet together in a dedicated release.
- Bumped: Gson 2.10.1 → 2.14.0, jackson-databind 2.16.1 → 2.19.0,
  SLF4J 2.0.9 → 2.0.18, Tycho 5.0.1 → 5.0.3.

## Verification

- Full reactor: **1169 tests, 0 failures, 0 errors** (1165 baseline + 4 new sentinel
  tests; 5 skips unchanged).
- Live proof on the built product: all **43 tools** answer with names identical to
  v2.7.1; `compose_method` plan → apply → undo round-trip leaves the tree
  byte-identical; experience-store record → recall → refresh round-trip clean;
  Studio deploy smoke green (managed endpoints healthy, guard + recall hooks attested,
  the #5188-patched search answering live). Studio suite: 174/0.
