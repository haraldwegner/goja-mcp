# JAWATA v2.8.1 — dogfood polish on the new platform

Four findings from pointing v2.8.0 at its own codebase the hour it shipped — same-day
fixes, same tradition as v2.7.1. No platform change; the Eclipse 2026-06 re-base and
the #5188 patch are untouched.

## Fixed

- **`find_references` honors its own documentation.** The front door advertised a
  `query` parameter that no delegate consumed (calls failed with INVALID_PARAMETER),
  while hiding the FQN form the underlying tools have supported since v1.8.0. The
  schema now declares `symbol` + `scope`, `query` works as an alias for `symbol`
  (an explicit `symbol` wins), and the description says plainly that
  `method_references` finds `Foo::bar` *expressions* — call sites live in
  `get_call_hierarchy(direction=incoming)`.
- **`search_symbols` ranks the project's own code first.** A Class search for
  `*Tool` on jawata-mcp itself listed `JavacTool`/`JShellTool` and friends above
  the project's classes — the bounded search window filled with JRE types before
  any source hit was fetched. The tool now queries a SOURCES-only scope first and
  tops up from the full classpath; binary hits are flagged `binary: true` and point
  at the readable archive (JAR/JMOD/jrt) instead of an opaque workspace handle.
- **`find_duplicate_code` is paged.** The self-scan returned 105 clone groups as one
  68 KB response. Now: `limit` (default 20) / `offset`, plus `summary:true` for
  counts only — `groupCount`/`instanceCount` always reflect the full set. Same
  shape `compile_workspace` got in v2.7.1.

## Added

- **`find_quality_issue(excludePaths=[...])`** — findings (and conflicts) whose file
  path contains any entry are dropped before counts, summary, baseline and
  pagination. Built for vendored code: the fowler sweep on jawata-mcp was flooded
  by the 12,378-line Eclipse `ProblemReporter` copy that ships the #5188 patch.

## Verification

- Full reactor: **1179 tests, 0 failures, 0 errors** (1169 baseline + 10 new; 5 skips
  unchanged).

## Compatibility

- The `goja.*`/`GOJA_*` legacy fallbacks are still in place; their removal is a
  recorded obligation for the next feature release (see the Sprint 22d spec).
