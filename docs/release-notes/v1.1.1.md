# goja-mcp v1.1.1 — generic category surface (Sprint 16b/A, completed)

Completes Stream A of Sprint 16b: the deeper generic consolidation. The loaded MCP
surface drops from **65 → 39 tools** (cumulative **85 → 39** since v1.0), clearing the
binding client cap (Cursor sends only the first ~40 MCP tools to the model) while the full
capability stays intact — the narrow tools remain in-package as the delegated
implementations behind each parametric category.

## New generic front doors

Read-only inspection collapsed into two broad categories:
- **`analyze(kind)`** ← `analyze_file`/`analyze_type`/`analyze_method`/`analyze_javadocs`/
  `analyze_naming`/`analyze_nullness`/`analyze_change_impact`/`analyze_control_flow`/
  `analyze_data_flow` (9→1). For javadocs/naming/nullness pass the analyzer's own variant
  as `subkind`.
- **`inspect(kind)`** ← `get_type_hierarchy`/`get_document_symbols`/`get_type_members`/
  `get_classpath_info`/`get_project_structure`/`get_type_usage_summary`/
  `get_complexity_metrics`/`get_dependency_graph`/`get_di_registrations` (9→1).

Action families collapsed into verbs:
- **`find_references(kind)`** ← references / implementations / method_references (3→1).
- **`refactoring(action)`** ← `apply_refactoring`/`undo_refactoring`/`inspect_refactoring` (3→1).
- **`project(action)`** ← `list_projects`/`add_project`/`remove_project` (3→1; `load_project` stays).
- **`quick_fix(action)`** ← `suggest_imports`/`get_quick_fixes`/`apply_quick_fix` (3→1).
- **`dependency(action)`** ← `add_dependency`/`update_dependency`/`find_unused_dependencies` (3→1).

Together with v1.1.0's `get_at_position` / `get_call_hierarchy` / `extract` / `inline` /
`move` / `move_in_hierarchy` / `generate`, the surface is now category-shaped: a compact,
always-loaded front door over the full (and growing) Java capability. Sprint 17 (Fowler) /
20 (SOLID) add **kinds** to the catalogs, not tools.

## Utilization
- **Cursor:** the ~39-tool surface now fits under the 40-tool cap — no silent drops.
- **Claude Code:** already loaded fully via the deploy's `alwaysLoad` (v1.1.0); unchanged.

## Compatibility
Collapsed narrow tools are no longer registered; use the category front doors. Each kind's
behaviour and output are unchanged (they delegate to the same implementations). Read-only
front doors (`analyze`, `inspect`, `find_references`, `get_*`) carry `readOnlyHint`.

## Verification
Full reactor `mvn clean verify` green; new parametric-routing tests (parity vs the narrow
delegate per kind/action); no behaviour regression.
