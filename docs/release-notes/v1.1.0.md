# goja-mcp v1.1.0 — the abstraction layer (Sprint 16b)

The metric is **utilization, not parsimony**: make the full, growing Java capability
*mandatory-loaded and reached-for* instead of deferred-and-skipped. v1.1.0 is the
architectural foundation the content sprints (17 Fowler → 21 experience store) build on.

## Highlights

### A compact, category-shaped front door (Stream A)
The MCP surface is collapsed from **85 → 65 tools** by folding families of narrow tools
into parametric category tools. The narrow classes remain in-package as the delegated
implementations — capability is unchanged, the *loaded surface* shrinks.

- **`get_at_position(kind)`** — one read-only front door over nine position lookups:
  `type` · `method` · `field` · `hover` · `javadoc` · `signature` · `enclosing` ·
  `super` · `symbol`.
- **`get_call_hierarchy(direction)`** — `incoming` (callers) · `outgoing` (callees).
- **`extract(kind)`** — `method` · `variable` · `constant` · `interface`.
- **`inline(kind)`** — `method` · `variable`.
- **`move(kind)`** — `class` · `package`; **`move_in_hierarchy(direction)`** — `up` · `down`.
- **`generate(kind)`** — `constructor` · `getters_setters` · `equals_hashcode` ·
  `tostring` · `test_skeleton` · `override_methods`.

All apply tools keep the behaviour-preserving apply/undo contract
(`filesModified` / `diff` / `undoChangeId` / `summary`).

### The domain spine — capability grows by registering a kind (Stream D)
`org.goja.mcp.domain` introduces GOJA's own engineering abstraction:
`Finding` (the canonical smell value type) + `Findings` renderer + `Detector` +
`DetectorCatalog` + `IGojaService`. **`find_quality_issue` is now a *projection* of the
detector catalog** — its `kind` enum and dispatch come from the registry. A detector
registered in the catalog appears in the tool and dispatches with **no edit to the tool**.
This is what Sprint 17 (Fowler) / 20 (SOLID) extend: *add a kind, not a tool or a service.*

### Always-load, deploy-owned (Stream C — goja-studio)
Reality check: a server cannot force a client to load schemas — deferral is the client's
call. GOJA now uses every lever it controls:
- Claude Code (v2.1.121+): the deploy marks the GOJA server `alwaysLoad`, so the whole
  surface loads upfront and never defers — **regardless of count**.
- The deploy writes the "use GOJA, not grep" rule block into each client's
  **global / always-loaded** instruction file (e.g. `~/.claude/CLAUDE.md`), not just the
  config-sibling — so the rule survives deferral and the hand-maintenance loop is closed.

### Single-service gateway (Stream B — goja-studio, opt-in)
A blocking in-process gateway lets the client see **one** `goja` MCP service that routes
each call to the owning per-workspace resident (which stays an isolated JVM — no merge,
no heap concentration). Routing: `tools/list`/`initialize` → any ready resident;
`tools/call` → the resident owning the call's `filePath` (longest prefix), then
`projectKey`. **Off by default** (`gateway_enabled`); the per-workspace deploy is unchanged
until you opt in.

## Utilization status (honest)
- **Claude Code: solved.** `alwaysLoad` forces the full 65-tool surface to load upfront —
  no deferral at any count.
- **Cursor: improved, not yet fully under the cap.** Cursor sends only the first ~40 MCP
  tools to the model; at 65 the surface still exceeds that. Further collapse of the
  `analyze_*` / `get_*` inspection families (a mechanical follow-on using the same
  pattern) is the path to ≤40, and the gateway reduces the *multi-workspace* total to one
  service. Until then, disable unneeded tools in Cursor, or run Claude Code.

## Compatibility
- Collapsed narrow tools are no longer registered; use the category front doors. Behaviour
  and output of each kind are unchanged (they delegate to the same implementations).
- `find_quality_issue` output and kinds are unchanged; it now resolves them via the catalog.

## Verification
Full reactor `mvn clean verify` green; new parametric + projection + gateway-routing tests;
no behaviour regression (the throws/catches query alias and all 8 quality kinds preserved).
