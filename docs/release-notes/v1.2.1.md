# goja-mcp v1.2.1 — Fowler smell precision tuning (Sprint 17 follow-up)

A precision pass on the v1.2.0 smell detectors, driven by **dogfooding the new
kinds against GOJA's own source**. v1.2.0's fixtures proved per-case correctness;
the live run on a 304-file corpus exposed two over-firing kinds — `long_method`
(237 findings) and `feature_envy` (179) — that would train an agent to ignore the
tool. Two root causes, both fixed:

## Test sources excluded by default
Smell scans now **skip test sources** (`src/test/`, `*.tests/` bundles) unless the
new `includeTests: true` flag is set. Test code legitimately has long methods and
methods that hammer their subject-under-test (a test "envies" the class it
exercises), so it was the bulk of the noise. The exclusion check runs on the path
**relative to the project root**, so a fixture that merely lives under a
`*.tests/` directory is not mis-classified.

## Tighter triggers
- **`long_method`** — default LOC cutoff raised **40 → 60** and the cyclomatic
  secondary trigger **10 → 15** (40 LOC / CC 11–14 methods are common on real code
  and not individually actionable). Lower `threshold` or pass `filePath` to scope
  tighter.
- **`feature_envy`** — the envied type must now beat the method's own type by a
  margin of **≥ 2** accesses (was "strictly greater"), so a method that barely
  edges its own type is no longer flagged.

## Net effect (measured on GOJA's own 304-file source)
- `feature_envy`: **179 → 27** — test-source exclusion alone removed 150 findings
  (test methods envying their subject-under-test), the margin trimmed 2 more.
- `long_method`: **237 → ~122** — long methods are an inherently populous smell;
  the higher default surfaces the worst offenders, and `filePath`/`threshold` scope
  the rest.
- Well-tuned kinds unchanged: `god_class` (3 — exactly the real central classes),
  `incomplete_delegation` (12 plausible scans).

All 18 remain `find_quality_issue` kinds — tool count still **39**, enum still **26**.

## Compatibility
Additive + behaviour-narrowing for two kinds. `includeTests` restores the v1.2.0
breadth where wanted. No API/tool-count change.
